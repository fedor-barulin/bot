# Enterprise RAG AI Support Agent (Телеком AI-Ассистент)

## 📌 Описание проекта

Этот проект представляет собой Enterprise-уровня RAG (Retrieval-Augmented Generation) систему, разработанную специально для закрытой корпоративной сети оператора связи. 

Главная цель системы — автоматизация и помощь сотрудникам в работе с внутренней базой знаний (инструкции, решения типичных проблем, процедуры, технические детали услуг связи), функционируя как опытный AI-инженер технической поддержки. Система отвечает строго на основе загруженных документов (pdf-чанков), не выдумывает факты и обязательно предоставляет ссылки на источники из базы знаний.

### Ключевые возможности и особенности архитектуры:
- **Query Understanding**: Анализ интента (намерения), затронутого сервиса и типа проблемы из вопроса сотрудника.
- **Knowledge Routing**: Эвристическая маршрутизация запроса в нужный раздел базы знаний для уменьшения шума поиска.
- **Query Rewriting**: Автоматическое перефразирование вопроса пользователя в несколько оптимальных вариантов с помощью LLM для максимизации результата поиска.
- **Hybrid Search**: Гибридный векторный поиск (через `Qdrant` и мультиязычную эмбеддинг модель `BAAI/bge-m3`).
- **Cross-Encoder Reranking**: Умный реранжинг (пересортировка) результатов поиска для выбора ТОП-5 наиболее релевантных чанков (через `BAAI/bge-reranker-large`).
- **Reasoning Engine & Anti-Hallucination**: LLM обрабатывает сжатый контекст и формирует профессиональный ответ в строгом стиле инженера (краткое объяснение, возможные причины, шаги диагностики, решение). Если необходимого факта нет — система выдает строго *"Информация отсутствует в базе знаний"*. Встроен также дополнительный шаг валидации сгенерированного ответа.
- **Интеграция с LLM**: Поддерживаются бесплатные API-вызовы через пакет `g4f` — нет необходимости поднимать собственные тяжелые локальные модели для старта разработки.
- **Собственный интерфейс**: React-чат с отображением источников, историей, возможностью копирования и кнопками обратной связи (👍/👎).
- **Аналитика и кеширование**: Память популярных запросов через Redis и логирование действий пользователя для улучшений.

## 🛠 Технологический стек

- **Backend**: Python 3.14, FastAPI, Pydantic, Redis (кеш).
- **Vector DB & Search**: Qdrant, SentenceTransformers (`bge-m3`), FlagEmbedding (`bge-reranker-large`).
- **LLM Engine**: Использование различных LLM через `g4f` (GPT-4 / Llama 3 формат wrapper'а).
- **Frontend**: React, Axios.
- **Инфраструктура**: Docker, Docker Compose.

## 🚀 Подробная инструкция по запуску и тестированию

### Предварительные требования
На вашем ПК (или сервере) должны быть установлены:
- [Docker Desktop](https://www.docker.com/products/docker-desktop/) или Docker Engine с плагином `docker-compose`.
- [Node.js / npm](https://nodejs.org/en/) (для запуска Frontend интерфейса).
- [Python 3.14+](https://www.python.org/downloads/) (опционально, требуется для запуска скрипта загрузки базы знаний напрямую со своей хост-машины).

### Шаг 1: Развертывание базовой инфраструктуры (Бэкенд, Qdrant, Redis)
Откройте командную строку/терминал, перейдите в подпапку `infra/` внутри проекта и запустите контейнеры в фоновом режиме сборкой:
```bash
cd infra
docker-compose up -d --build
```
> **Что произойдет?** 
> - Скачаются образы и поднимется **Qdrant** (база векторов, порт `6333`).
> - Поднимется **Redis** (кеш-сервер, порт `6379`).
> - Соберется и запустится **Backend API** на FastAPI (порт `8000`). Скачивание ML зависимостей займет время.

### Шаг 2: Настройка LLM Engine (via g4f)
В проекте применим клиент `g4f` — он автоматически связывается с бесплатными провайдерами LLM для предоставления ответов (вам **не нужно** устанавливать и поднимать локальную модель Llama или задействовать свои GPU ресурсы для тестового прогона).

Все зависимости `g4f` уже установлены внутри Docker-образа бэкенда и готовы работать из коробки!

### Шаг 3: Наполнение базы знаний (векторизация чанков)
Пока Qdrant и Backend работают, вам нужно "наполнить" векторную базу ваших документов. Для демонстрации создан специальный скрипт `data/example_upload.py`.
Откройте новый терминал в корневой папке всего проекта:

1. Создайте виртуальное окружение (рекомендуется) и установите зависимости скрипта:
```bash
pip install qdrant-client sentence-transformers
```
2. Выполните скрипт первичной логики:
```bash
python data/example_upload.py
```
> **Что произойдет?** Скрипт подключится к Qdrant, создаст векторную коллекцию `knowledge_base`, переведет тестовый чанк (структурированный кусок текста про мобильный интернет из PDF) с помощью модели `bge-m3` в векторы и загрузит в БД.

*(По такому же образцу вы можете спарсить все ваши сотни созданных PDF чанков!)*

### Шаг 4: Запуск пользовательского интерфейса (Frontend чат)
Теперь запустим визуальную оболочку, в которой работают сотрудники.
Откройте отдельное окно терминала и перейдите в папку фронтенд приложения:
```bash
cd frontend/chat_ui
npm install
npm start
```
Ваш дефолтный браузер должен автоматически открыть страницу `http://localhost:3000`.

### Шаг 5: Проверка работы (Демонстрация Pipeline)
1. Откройте чат оператора в браузере.
2. Введите пользовательский или тестовый вопрос. Например: *"почему не работает интернет"*, или *"как подключить мобильный интернет?"*
3. **Что происходит под капотом во время вашего запроса:**
   - FastAPI проверяет Redis кэш, чтобы моментально отдать ответ на частые вопросы.
   - Запрос анализируется на Интент (`troubleshooting`), Сервис (`mobile internet`) и Проблему.
   - Вопрос переписывается на похожие ключевые фразы через LLM (для полноты поиска).
   - Qdrant осуществляет гибридный векторный поиск по вашей базе чанков.
   - Найденные документы сортируются через тяжелую ML модель (Cross-Encoder Reranker).
   - Формируется контекст и выбранная через `g4f` LLM генерирует технический ответ.
   - Сверх-строгий Answer Validator отбраковывает галлюцинации и возвращает финальный ответ с цитированием (Раздел документа и т.д).

## 📂 Структура системы

* `backend/` — Микросервисная архитектура RAG пайплайна (Анализ, Маршрутизация, Реранкинг, LLM-reasoning, Анти-Галлюцинации, API FastAPI).
* `frontend/` — Реализация Enterprise React UI Чата со ссылками на источники.
* `data/` — Инструменты и скрипты для загрузки готовых (parsed) PDF чанков.
* `infra/` — Dockerfile бэкенда и `docker-compose.yml` для автоматизации развертывания системы.
