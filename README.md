# Мотив AI — Корпоративный RAG-ассистент

Внутренний AI-помощник для сотрудников оператора связи **Мотив**. Отвечает строго на основе загруженных внутренних документов, не выдумывает факты и всегда ссылается на источник из базы знаний. Поддерживает многоходовой диалог с учётом истории переписки.

---

## Технологический стек

| Компонент | Решение |
|---|---|
| **Backend** | Python 3.12, FastAPI, Uvicorn |
| **Embeddings** | `BAAI/bge-m3` via SentenceTransformers |
| **Vector DB** | Qdrant (локальный disk-режим, `data/qdrant_storage/`) |
| **LLM** | `g4f` — бесплатные провайдеры, без GPU и API-ключей |
| **Frontend** | React 18, Axios, фирменный дизайн Мотив |
| **Cache** | LRU in-memory (результаты векторного поиска) |

---

| Воркфлоу | Команда | Порт |
|---|---|---|
| **Start Backend** | `cd backend && python -m uvicorn main:app --host localhost --port 8000` | 8000 |
| **Start application** | `cd frontend/chat_ui && npm start` | 5000 |

Запросы `/chat` из фронтенда проксируются на `http://localhost:8000` через поле `proxy` в `package.json`.

> **Первый запуск** занимает 20–30 секунд — бэкенд загружает модель `BAAI/bge-m3` и прогревает PyTorch. «Application startup complete» появляется после готовности модели.

## Как работает RAG-пайплайн

При каждом запросе система проходит 4 шага:

```
Вопрос + история диалога
         │
         ▼
1. Векторный поиск      — BAAI/bge-m3 кодирует вопрос → top-3 чанка из Qdrant
   (LRU-кеш)              повторный вопрос → мгновенно из кеша
         │
         ▼
2. Сборка контекста     — текст чанков, лимит 4500 символов
         │
         ▼
3. Вызов LLM (g4f)      — системный промпт-дерево (Шаги A→B→C) + контекст + история
   max_tokens=1000         если ответ не в базе → «Информация отсутствует в базе знаний»
         │
         ▼
4. Очистка ответа       — убрать фразы-паразиты, реклама g4f-провайдера, нормализация
         │
         ▼
Ответ + ссылки на источники (документ, раздел, страница)
```

### Логика системного промпта (дерево решений)

- **Шаг A** — Если вопрос не про телеком → вежливый отказ, стоп.
- **Шаг B** — Если вопрос неоднозначен → уточняющий вопрос с вариантами, стоп. Допускается использование общих телеком-знаний только для понимания смысла вопроса.
- **Шаг C** — Ответ строго на основе контекста из базы. Каждый факт должен быть явно взят из чанка. Запрет на домысливание.

Фирменный дизайн Мотив: оранжевый бренд-цвет `#f37021`, тёмный фон заголовка `#21242c`, пузырь пользователя `#e8734a`.

### Быстрые ответы (Quick Replies)

При наличии вопроса в ответе бота система автоматически парсит варианты и отображает их как кнопки-чипы над полем ввода. Алгоритм:
- Список элементов (`- вариант 1`, `- вариант 2`) → отдельные чипы
- Опции через «или» → разбиваются на чипы
- По умолчанию — стандартные подсказки из `SUGGESTED`

### Стандартные подсказки

```js
const SUGGESTED = [
  'Как настроить мобильный интернет?',
  'Как сменить тариф?',
  'Как связаться с поддержкой?',
  'Дополнительные услуги в Свой тариф',
];
```

## Наполнение базы знаний

### Формат JSON (рекомендуется)

Массив объектов, каждый объект — один чанк:

```json
[
  {
    "title": "Мобильный Интернет",
    "source": "Мобильный Интернет.pdf",
    "section": "Общая информация",
    "chunk_id": 1,
    "text": "Обязательная услуга для доступа в Интернет по технологии LTE (4G)...",
    "page": 1,
    "tags": ["internet", "activation"],
    "language": "ru"
  }
]
```

| Поле | Обязательное | Описание |
|---|---|---|
| `title` | да | Название документа (без `.pdf`) |
| `source` | нет | Имя файла-источника |
| `section` | да | Раздел внутри документа |
| `chunk_id` | нет | Номер чанка (генерируется автоматически) |
| `text` | да | Текст чанка — именно он векторизуется |
| `page` | да | Номер страницы в исходном PDF |
| `tags` | нет | Массив строк-тегов |
| `language` | нет | Код языка (не используется системой) |

### Формат Markdown

```markdown
### [Название файла.pdf] Раздел: Название раздела (Стр. N)
**Теги:** тег1, тег2

Текст чанка...

---
```

### Загрузка через API (бэкенд не нужно останавливать)

```bash
# Загрузить JSON или Markdown
curl -X POST http://localhost:8000/admin/upload -F "file=@my_data.json"

# Сначала очистить базу, потом загрузить новое
curl -X POST http://localhost:8000/admin/clear
curl -X POST http://localhost:8000/admin/upload -F "file=@my_data.json"
```

### Загрузка через скрипт (требует остановки бэкенда)

Бэкенд и скрипт не могут одновременно держать lock на локальное хранилище Qdrant.

```bash
python data/upload_markdown.py data/my_file.json
python data/upload_markdown.py data/my_file.md
```

## API

### `POST /chat`

**Запрос:**
```json
{
  "question": "как подключить мобильный интернет?",
  "history": [
    {"role": "user", "content": "привет"},
    {"role": "assistant", "content": "Здравствуйте! Чем могу помочь?"}
  ]
}
```

**Ответ:**
```json
{
  "answer": "Для подключения мобильного интернета необходима USIM-карта...",
  "sources": [
    {"title": "Мобильный Интернет", "section": "Общая информация", "page": 1}
  ]
}
```

Поле `history` опционально.

### `POST /admin/upload`

Загружает `.json` или `.md` файл в базу знаний. Принимает `multipart/form-data` с полем `file`.

### `POST /admin/clear`

Полностью очищает коллекцию `knowledge_base` в Qdrant.

## Известные ограничения

- **g4f** использует бесплатные провайдеры — время ответа LLM зависит от их доступности (обычно 10–40 сек). Провайдер может добавлять рекламный текст — он автоматически вырезается в `_clean_response`.
- **Только CPU** — модель `BAAI/bge-m3` запускается без GPU; повторные запросы быстрые благодаря LRU-кешу.
- **Локальный Qdrant** — одновременно только один процесс может держать lock на хранилище `data/qdrant_storage/`. Используйте API-эндпоинты `/admin/upload` и `/admin/clear` при работающем бэкенде.
- **OCR-шум в чанках** — чанки из PDF содержат артефакты распознавания (навигационные элементы интерфейса). Для лучшего качества рекомендуется предварительно чистить текст перед загрузкой.
